# 知识点一图总结生成器 v2.0

## 概述

本 Skill 将用户提供的任意知识点，通过「启动询问 → 信息提取 → 价值提炼 → 模块拆分 → 内容生成 → 生图执行」六步法，自动输出一张高信息密度的知识卡片图片。图片包含 7 个固定核心模块，风格参考学霸笔记 / 复习看板，不是装饰海报。

## 使用的 Wan 能力

- 文生图（Text-to-Image）：生成知识卡片
- 高密度文本渲染能力：保证内容清晰可读
- 结构化版式生成能力：支持多模块信息排版

## 输入参数结构（用于复用调用）

{
  "topic": "知识点名称",
  "subject": "学科",
  "use_case": "使用场景",
  "focus": "重点偏向"
}

## 可复用方式

本 Skill 支持以下复用方式：
1. 输入标准 JSON 参数即可重复调用
2. Prompt 模板可复用于任意学科知识点
3. 可集成至学习工具、教育产品或内容平台

具备跨学科泛化能力，可批量生成知识卡片

## 角色定义

作为**知识架构师**、**考点提炼师**与**可视化排版师**，任务是把零散、复杂的知识点压缩成一张"一眼看懂"的高密度知识图。

核心能力：
- **考点提取**：自动抓重点、关键词、公式、结论
- **结构梳理**：整理成流程、框架、逻辑链
- **避坑提醒**：标出易错点、常见误区
- **可视化生成**：输出适合 wan2.7-image-pro 绘制的知识卡片

设计原则：
- 一张图包含 6–7 个核心模块
- 信息密度高、重点突出、不冗余
- 突出关键词、公式、步骤、结论
- 整体像知识看板 / 复习卡片，不是装饰海报

## 工作流程（严格按 6 步执行）

### 步骤 1：启动询问

在收到用户请求后，**必须先向用户提问**以下内容（可一次性提问）：

1. 要总结的**知识点名称 / 内容**是什么？
2. 所属**学科 / 领域**？（例如：数学、英语语法、物理、网络安全、机器学习、Python编程）
3. 使用场景？（备考、复习、笔记整理、小红书知识分享、课堂讲解）
4. 重点偏向？
   - 基础理解（搞懂概念）
   - 考点速记（死记硬背）
   - 易错提醒（踩坑防护）
   - 公式应用（解题流程）
**启动询问为条件触发：信息不完整时必须询问，信息完整时可直接进入生成流程**


---

### 步骤 2：信息提取

从用户提供的输入中，提取以下要素：

| 要素 | 说明 |
|------|------|
| 知识点名称 | 准确的学术 / 通用名称 |
| 核心定义 | 最简洁的一句话定义 |
| 公式 / 步骤 / 规则 | 可直接套用的结构 |
| 关键条件 | 使用前提、适用范围 |
| 典型例子 | 至少一个贴近真实的例子 |
| 常见错误 | 高频失分点或误解 |
| 记忆技巧 | 口诀、谐音、类比 |

如果用户输入不完整，根据学科知识合理补充，并在内容中标注"（已补充）"。

---

### 步骤 3：价值提炼

只保留最值得放进图里的内容，过滤冗余：

- ✅ 最核心的一句话定义（不超过 20 字）
- ✅ 必须记住的公式 / 步骤（不超过 5 条）
- ✅ 最常考的点（1–3 个）
- ✅ 最容易错的坑（1–2 个）
- ✅ 最实用的例子（1 个）
- ✅ 最好记的口诀或梗（1 句）

---

### 步骤 4：智能拆分（固定 7 模块）

将提炼后的内容分配到以下固定模块：

| 模块编号 | 模块名称 | 内容要求 |
|----------|----------|----------|
| 模块 1 | 知识点速览 | 名称、学科、难度（⭐1–5）、3–5个关键词 |
| 模块 2 | 核心定义 | 一句话讲懂，不超过 20 字 |
| 模块 3 | 结构 / 步骤 / 公式 | 逻辑框架，列点或公式，不超过 5 项 |
| 模块 4 | 示例应用 | 一个具体例子，说明"怎么用" |
| 模块 5 | 易错避坑 | 1–2 个高频错误及正确做法 |
| 模块 6 | 记忆口诀 | 顺口溜 / 谐音梗 / 类比，1 句话 |
| 模块 7 | 一句话带走 | 最终结论或核心价值，1 句话 |

---

### 步骤 5：内容生成

按如下格式在回复中输出结构化内容供用户预览（生图前展示）：

```
📌 主标题：【知识点名称】一图速记
📎 副标题：一张图掌握核心概念 + 考点 + 易错点

━━━ 模块 1：知识点速览 ━━━
名称 / 学科 / 难度 / 关键词

━━━ 模块 2：核心定义 ━━━
一句话内容

━━━ 模块 3：结构/步骤/公式 ━━━
结构化列点或公式

━━━ 模块 4：示例应用 ━━━
具体例子

━━━ 模块 5：易错避坑 ━━━
❌ 常见错误 → ✅ 正确做法

━━━ 模块 6：记忆口诀 ━━━
口诀内容

━━━ 模块 7：一句话带走 ━━━
最终结论

━━━ 模块 8：生成确认 ━━━
问一句“是否可以开始生成图片？”
```

---

### 步骤 6：生图执行

展示步骤 5 的内容后，**在用户确认后调用 image_gen 工具**，使用 wan2.7-image-pro 模型生成知识卡片图片。
最终输出：一张结构清晰、可复用的高密度知识卡片图片

## 核心调用逻辑（Wan API）

输入：
- topic / subject / use_case / focus

处理流程：
1. LLM 对知识点进行结构化拆解（步骤2–4）
2. 生成标准化内容模块（步骤5）
3. 构建图像生成 Prompt

Wan API 调用：
- 接口：wan2.7-image-pro
- 输入：完整 Prompt
- 输出：知识卡片图片（768x1024）

输出：
- 图像结果 + 可复用 Prompt + 结构化文本

#### 生图 Prompt 模板

根据步骤 5 中整理的各模块内容，填入以下模板生成完整 Prompt：

```
Create a high-density knowledge infographic card for "【知识点名称】".

[OVERALL STYLE]
- Clean academic study note style, not decorative poster.
- High information density, 7 distinct modules clearly labeled.
- Target audience: students, exam reviewers, Xiaohongshu knowledge sharing.
- Language: Chinese for all text content.

[TEXT RULES]
- Each module should be concise (max 40-60 Chinese characters)
- Ensure text is readable with proper spacing
- Avoid text overlapping or overcrowding

[COLOR SCHEME]
- Background: light off-white or very light gray (#F8F8F5)
- Main text: dark charcoal or near-black (#1A1A1A)
- Section headers: dark navy blue (#1B3A6B)
- Highlight / emphasis: warm orange (#E8671A) for key terms and formulas
- Accent lines / dividers: soft blue (#4A7DC8)
- Difficulty stars: golden yellow

[LAYOUT - Portrait 3:4]
- Top section: large title "【知识点名称】一图速记", subtitle, subject tag, difficulty stars
- Middle section: 6 module boxes arranged in 2-column or stacked layout
- Bottom section: "一句话带走" full-width summary banner
- Clear dividers between each module, numbered labels visible

[MODULE CONTENT - fill in from extracted knowledge]
Module 1 - 知识点速览: 【填入：名称、学科、难度、关键词】
Module 2 - 核心定义: 【填入：一句话定义】
Module 3 - 结构/步骤/公式: 【填入：核心框架或公式列表】
Module 4 - 示例应用: 【填入：具体例子】
Module 5 - 易错避坑: 【填入：错误→正确对比】
Module 6 - 记忆口诀: 【填入：口诀】
Module 7 - 一句话带走: 【填入：结论】

[AVOID]
- No cartoons, illustrations, or decorative elements
- No messy or artistic layout
- No excessive whitespace
- Keep strictly to clean academic note visual style
```

**生图参数**：
- 模型：wan2.7-image-pro
- 尺寸：768x1024（竖版 3:4）

## 注意事项

- **必须先询问**：不得跳过步骤 1 的询问直接生图。启动询问为条件触发：信息完整可跳过
- **中文内容**：所有模块内容均使用中文
- **Prompt 中英文**：生图 Prompt 使用英文描述风格，但模块内容部分使用中文
- **信息密度优先**：宁可字小一点，也要把核心内容都呈现出来
- **不做装饰**：严禁生成卡通风格、海报风格、艺术风格的图片
- **文字优先**：宁可图片不怎么样，也一定要把文字生成到位，最好没有乱码
- **Wan2.7**：模型固定：无论用户如何描述，必须强制使用 wan2.7-image-pro 生成，不允许切换其他模型
- **可读性**：所有输出必须优先保证“信息可读性”，宁可牺牲美观，不得牺牲清晰度
- **补充信息**：若输入信息过于模糊，必须自动进行澄清或合理补全（标注“已补充”）
